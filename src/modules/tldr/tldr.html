M-name


<script M-name="homepage-hero">include('modules/homepage/hero.html', 'homepage-hero')</script>

The M-name should be the same as the second argument in the include function

M-lazy


<script M-lazy="300" M-name="address">include('modules/address/address.html', 'address')</script>	
<script M-lazy M-name="footer">include('modules/footer/footer.html', 'footer')</script>

Modules that include the M-lazy attribute will not be loaded until they are within the specified amout of pixels from the bottom of the screen (default 100px) a negative value can be specified which will defer loading until the top of the module is above the bottom of the screen by that value

M-hash


<a M-hash="address" href="#">show address</a>
<script M-hash M-name="address">include('modules/address/address.html', 'address')</script>

Modules that include the M-hash attribute will not be loaded until corresponding anchor is clicked.

M-temp


<script M-temp M-name="blog">include('modules/blog/blog.html', 'blog')</script>	
<script M-temp="data/posts.json" M-name="blog">include('modules/blog/blog.html', 'blog')</script>

Modules that include the M-temp attribute will put through the doT.js template engine

Responsive module loading


<script M-name="homepage-hero">include('modules/homepage/hero[.size].html', 'homepage-hero', ["(mob)440", "440(tab)800", "800()"])</script>

by adding [.size] or [/size] to the url of the module, and by passing an array like ["(mob)440", "440(tab)800", "800()"] as an argument in the include, you can load different modules, depending on the width of the users device.

module screen size
modules/homepage/hero.mob.html <= 440px
modules/homepage/hero.tab.html > 440px and <= 800px
modules/homepage/hero.html > 800px

Responsive module loading


<script M-name="homepage-hero">include('modules/homepage/hero[.lang].html', 'homepage-hero', "", ["fi", "sv"])</script>

by adding [.lang] or [/lang] to the url of the module, and by passing an array like ["fi", "sv"] as an argument in the include, you can load different modules, depending on the language passed in the url, eg ?lang=fi or by setting the localStorage language variable to "fi"

module language
modules/homepage/hero.fi.html Finnish
modules/homepage/hero.sv.html Swedish
modules/homepage/hero.html default (for no language)
